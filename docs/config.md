# Configuration

All configuration lives in `config.yml`. The file is generated by `/setup` or copied from `config.yml.example`. A `private/*.yml` overlay deep-merges over it at runtime — real names, keys, and personalised prompts go there and are never committed.

## Identity

```yaml
user:
  name: "Ada"              # used in prompts and the brief header
  role_key: "ada"          # message label in the bot and dashboards
  profile: "developer"     # freelancer | developer | student | other
  background: "embedded dev, Turin"
  timezone: "Europe/Dublin"
```

## Goals

```yaml
goals:
  headline: "ship v1 by December"   # rides in every brief as the lead
  deadline: "2026-12-31"
```

## Brief

```yaml
brief:
  time: "09:00"
  priority_ordering:
    - cash_collection
    - cold_outreach
    - skills_study
  news_topics:
    - "cybersecurity vulnerability news"
    - "AI tools productivity"
```

Brief time is the launchd trigger — change it here, then reload:

```bash
launchctl unload ~/Library/LaunchAgents/com.nzt48.brief.plist
launchctl load  ~/Library/LaunchAgents/com.nzt48.brief.plist
```

## Modules

Both are off by default. Enable only what applies to you.

```yaml
modules:
  business:
    enabled: false
    offer: "hand-coded websites for local businesses"
    pipeline_file: "09-FINANCE/web-business-pipeline.md"
    lead_scoring:
      no_website: 40
      social_only_recent: 20
      phone_accessible: 15
      local_to_area: 10
    tiers: { A: 70, B: 45 }    # A → call · B → email · else archive

  learning:
    enabled: false
    platforms: ["TryHackMe", "HTB Academy"]
    progress_file: "06-LEARNING/platform-progress.md"
```

## Vault

```yaml
vault:
  path: "~/Documents/Notes"
  hot_files:
    hot_cache: "00-META/HOT-CACHE.md"     # always in every prompt
    priorities: "00-META/PRIORITIES.md"
  protected_prefixes:
    - "01-PROFILE/"
    - "02-GOALS/"
    - "00-META/HOT-CACHE.md"
    - "00-META/PRIORITIES.md"
```

Any path under `protected_prefixes` requires a Telegram tap before the agent can write. Extend this list freely.

## Private overlay

Create `private/euan.yml` (or any `private/*.yml`) with keys that override `config.yml`:

```yaml
user:
  name: "Euan"
telegram:
  allowed_ids: [123456789]
integrations:
  composio:
    enabled: true
    api_key: "sk-..."
```

The overlay merges recursively at boot. Nothing in `private/` is ever committed.
